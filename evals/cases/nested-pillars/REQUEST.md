# Nested Pillars evaluation

Use godplans to plan a monorepo with shared root guidance and an independently
governed `packages/web` application. The root scope owns repository and release
rules. The web scope overrides local UI guidance and excludes a root-routed
observe concern because the hosting platform owns package telemetry. Keep known
but unauthored privacy guidance discoverable through the local absent catalog.

Take recommended defaults. Produce the complete godplans artifact set under
`.godplans/`, including `PLAN.mdx` and its executable validator companion. Do
not build the repository.
