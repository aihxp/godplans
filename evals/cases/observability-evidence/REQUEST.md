# Observability evidence labels evaluation

Use godplans to plan observability for a public API service. A controlled
production-equivalent fire will exercise telemetry, paging, ownership, and a
runbook before launch. No real incident has happened yet. The plan must let the
controlled fire prove installation readiness without claiming operational
maturity or rewriting synthetic evidence as incident history.

Take recommended defaults. Produce the complete godplans artifact set under
`.godplans/`, including `PLAN.mdx` and its executable validator companion. Do
not build the service.
