# Observability plan

Write a thorough implementation plan for observability of a public API service.
A controlled production-equivalent fire will exercise telemetry, paging,
ownership, and a runbook before launch. No real incident has happened yet. The
plan must let the controlled fire prove installation readiness without claiming
operational maturity or presenting synthetic evidence as incident history.

Decide sensible defaults yourself rather than asking. Write your plan to
`PLAN.md`. Do not build the service.
